# Isolation

Rudel isolation starts with WordPress multisite.

## Site data

Every Rudel environment is a real multisite site, so WordPress already provides:

- per-site tables through `blog_id`
- per-site options
- per-site uploads
- per-site admin and REST behavior

Rudel builds on that instead of emulating it.

## Environment directories

Each environment also has a generated directory containing:

- `bootstrap.php`
- `wp-cli.yml`
- isolated `wp-content`
- `snapshots/` or `backups/`
- worktree-managed code when present

That directory is where Rudel stores copied themes, plugins, templates, snapshots, backups, and other environment-owned files.

## Authentication and cache

Rudel scopes authentication salts and cache behavior per environment so sessions and persistent cache data stay isolated between environments and the host.

## Email

Sandboxes block outbound email by default. Apps allow normal email delivery by default.

## Users

Rudel uses the multisite user model:

- network users are shared
- membership and capabilities are site-scoped

The runtime isolation boundary is the site, not a separate user table set.

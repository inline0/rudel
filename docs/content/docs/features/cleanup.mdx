# Cleanup

Sandboxes are disposable by design. Rudel provides cleanup mechanisms to prevent disk usage from growing unbounded.

## Running cleanup

Remove all sandboxes older than the configured maximum age:

```bash
wp rudel cleanup
```

To see what would be removed without actually deleting anything:

```bash
wp rudel cleanup --dry-run
```

You can override the configured max age for a single run:

```bash
wp rudel cleanup --max-age-days=7 --dry-run
```

## Configuration

Cleanup behavior is controlled by the Rudel configuration file (`rudel-config.json` in the `wp-content` directory):

| Setting | Description | Default |
|---|---|---|
| `max_sandboxes` | Maximum number of active sandboxes | 0 (no limit) |
| `max_age_days` | Sandboxes older than this are eligible for cleanup | 0 (disabled) |
| `max_disk_mb` | Maximum total disk usage for all sandboxes in MB | 0 (no limit) |

The `max_age_days` setting is enforced when you run `wp rudel cleanup`. The `max_sandboxes` and `max_disk_mb` limits are enforced at creation time, preventing new sandboxes from being created when the limit is reached.

If `max_age_days` is 0 and no `--max-age-days` override is provided, cleanup returns immediately with no removals.

## Merge-based cleanup

If you use Rudel's [git integration](/docs/features/git), sandboxes can be cleaned up automatically when their branches are merged. This is separate from age-based cleanup:

```bash
wp rudel cleanup --merged
```

This checks each sandbox that has git worktrees. If all of a sandbox's branches have been merged into the default branch (usually `main`), the sandbox is destroyed, the worktrees are removed, and the branches are deleted.

Preview first with `--dry-run`:

```bash
wp rudel cleanup --merged --dry-run
```

This is the cleanest way to manage sandbox lifecycles when working with git: create a sandbox, do the work, merge the PR, and the sandbox disappears on the next cleanup run.

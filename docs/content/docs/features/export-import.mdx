# Export & Import

Sandboxes can be exported as zip archives and imported on any Rudel installation. This makes it straightforward to share environments, create backups, or move sandboxes between servers.

## Exporting

```bash
wp rudel export my-sandbox-a1b2 --output=/tmp/sandbox.zip
```

The archive includes the sandbox's database, `wp-content` directory, and metadata. Everything needed to recreate the sandbox elsewhere.

## Importing

```bash
wp rudel import /tmp/sandbox.zip --name=imported-sandbox
```

This creates a new sandbox from the archive. The imported sandbox gets a fresh ID, and all internal URLs and table prefixes are rewritten to match. It's ready to use immediately.

## Typical workflows

**Sharing with a colleague.** Export a sandbox that reproduces a bug, send the zip, and they can import it to see the exact same state.

**Backups.** Periodically export important sandboxes for safekeeping.

**CI/CD.** Include an exported sandbox in your repository and import it during CI for consistent test environments.

**Migration.** Move sandboxes between staging and production WordPress installations.

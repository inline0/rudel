# Snapshots and Backups

Rudel uses two recovery-point models:

- sandboxes use **snapshots**
- apps use **backups**

## Sandbox snapshots

Create a snapshot:

```bash
wp rudel snapshot alpha-1234 --name=before-update
```

Restore it:

```bash
wp rudel restore alpha-1234 --snapshot=before-update --force
```

Snapshots capture the sandbox site state and its environment-owned content so you can roll back risky work quickly.

## App backups

Create a backup:

```bash
wp rudel app backup demo-1234 --name=baseline
```

Restore it:

```bash
wp rudel app restore demo-1234 --backup=baseline --force
```

App deploys also integrate with backups so recovery is built into the deploy workflow.

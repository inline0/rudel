# Templates

Templates let you freeze a sandbox's state and reuse it as a starting point for new sandboxes. Instead of configuring the same theme, plugins, and content every time, set it up once and save it as a template.

## Saving a template

```bash
wp rudel template save my-sandbox-a1b2 --name=starter
```

This copies the sandbox's database and `wp-content` into a named template. You can optionally add a description:

```bash
wp rudel template save my-sandbox-a1b2 --name=starter --description="WordPress with WooCommerce and Storefront theme"
```

## Creating from a template

```bash
wp rudel create --name="new-project" --template=starter
```

The new sandbox starts with the template's database and content. All URLs and table prefixes are rewritten for the new sandbox automatically.

## Managing templates

List all available templates:

```bash
wp rudel template list
```

Delete a template you no longer need:

```bash
wp rudel template delete starter --force
```

## Typical workflows

**Standardized environments.** Set up your preferred theme, plugins, and settings once, then use the template for every new sandbox.

**Team onboarding.** Share a template so every developer starts from the same baseline.

**Testing baselines.** Template a known-good state and create sandboxes from it for repeatable, consistent testing.

**Client demos.** Pre-configure templates for different client scenarios and spin up fresh sandboxes on demand.

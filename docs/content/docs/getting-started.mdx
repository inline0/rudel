# Getting Started

## Installation

### Via Composer (Recommended)

```bash
composer require rudel/rudel
```

```bash
wp plugin install rudel.zip --activate
```

## Setup

On activation, Rudel adds a single line to your `wp-config.php`:

```php
require_once __DIR__ . '/wp-content/plugins/rudel/bootstrap.php';
```

This bootstrap file runs before `wp-settings.php` and handles all sandbox detection and isolation. When no sandbox is active, it does nothing and WordPress boots normally with zero overhead.

You can verify everything is in place with:

```bash
wp rudel status
```

## Configuration

Rudel works out of the box with sensible defaults. If you need to customize the CLI command name or the URL path prefix, define these constants in `wp-config.php` before the plugin loads:

| Constant | Default | Description |
|---|---|---|
| `RUDEL_CLI_COMMAND` | `rudel` | WP-CLI command name. Set to e.g. `sandbox` for white-label use. |
| `RUDEL_PATH_PREFIX` | `__rudel` | URL path prefix for sandbox access. Change to customize the URL structure. |

## Create your first sandbox

```bash
wp rudel create --name="my-first-sandbox"
```

This creates a blank sandbox with its own database and an empty `wp-content` directory. The output includes the sandbox ID, filesystem path, and access URL.

To work with the sandbox via WP-CLI, just `cd` into its directory:

```bash
cd /path/to/sandboxes/my-first-sandbox-a1b2
wp post list
```

Any `wp` command run from within the sandbox directory is automatically scoped to that sandbox's database and content, thanks to the auto-generated `wp-cli.yml`.

If you'd rather start with a copy of your host site, use `--clone-all`:

```bash
wp rudel create --name="full-clone" --clone-all
```

This clones the host database along with all themes, plugins, and uploads. See [Creating Sandboxes](/docs/sandboxes/creating) for all the available options.

## Accessing sandboxes in the browser

Sandboxes are accessible via a path prefix that works without any DNS or server configuration:

```
example.com/__rudel/my-first-sandbox-a1b2/
```

When you visit a sandbox URL, Rudel automatically sets a browser cookie so that subsequent requests, including `/wp-admin/`, stay in the sandbox context. You can browse the sandbox frontend, log into the admin, and manage content as if it were a standalone site.

To switch back to the host, append `?adminExit` to any URL:

```
example.com/?adminExit
```

Sandboxes can also be accessed via subdomains (`sandbox-id.example.com`, requires wildcard DNS), the `X-Rudel-Sandbox` header (for programmatic access), or the `rudel_sandbox` cookie directly.

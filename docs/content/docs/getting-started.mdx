# Getting Started

## Prerequisites

Rudel runs on a WordPress multisite network configured for subdomains.

Before installing Rudel, make sure:

- WordPress multisite is enabled
- the network uses subdomains, not subdirectories
- you can write to `wp-config.php`

## Install Rudel

### Via Composer

```bash
composer require rudel/rudel
wp plugin activate rudel
```

### Via zip

```bash
wp plugin install rudel.zip --activate
```

## Verify the runtime install

```bash
wp rudel status
```

That confirms the bootstrap line is installed and Rudel can manage the network runtime.

## Create a sandbox

```bash
wp rudel create --name=alpha
```

Find the generated site URL:

```bash
wp site list --fields=blog_id,url
```

Run commands against it:

```bash
wp --url=http://alpha.localhost option get siteurl
```

Or work from the generated environment directory:

```bash
cd /path/to/wp-content/rudel-environments/alpha-1234
wp option get siteurl
```

## Create an app

```bash
wp rudel app create --name=demo --domain=demo.example.test
```

Apps are also real multisite sites. The `--domain` value is stored in Rudel's app metadata, while the runtime site itself is created as a real multisite site and listed through `wp site list`.

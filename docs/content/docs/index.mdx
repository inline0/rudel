# Introduction

Rudel lets you create fully isolated WordPress sandbox environments from within an existing WordPress installation. Each sandbox gets its own database, its own `wp-content` directory, and its own WP-CLI scope. Create one in under two seconds, use it however you like, and destroy it when you're done.

## Why Rudel?

WordPress doesn't have a good answer for disposable environments. Testing a plugin change on a live site is risky. Spinning up Docker or a local dev stack is heavy. Staging environments are expensive to maintain and slow to provision.

If you're working with AI coding agents, the problem is even sharper. Agents need a safe, scoped WordPress context where they can make changes, run commands, and iterate without risk to real data. That doesn't exist out of the box.

Rudel solves this with a single plugin. Install it, create a sandbox, and you have a fully isolated WordPress environment that shares the host's core files but nothing else. No Docker, no DNS, no separate database server.

## How it works

On activation, Rudel adds one line to `wp-config.php` that loads a bootstrap file before WordPress boots. This bootstrap detects whether the incoming request is targeting a sandbox, and if so, rewires all the relevant WordPress constants to point to that sandbox's isolated environment. When no sandbox is active, the bootstrap does nothing and WordPress boots normally.

Each sandbox lives in its own directory under `wp-content/rudel-sandboxes/`. The directory contains a `wp-cli.yml` that auto-scopes CLI commands, a `bootstrap.php` that sets the sandbox's constants, and an isolated `wp-content/` with its own themes, plugins, and uploads.

The database layer is configurable. By default, sandboxes use MySQL with an isolated table prefix in the same database as the host. You can also use SQLite for portable, file-based isolation, or create sandboxes as multisite sub-sites on multisite installations.

## Key features

Rudel is designed around a few principles: sandboxes should be instant to create, fully isolated from each other and the host, and disposable without cleanup headaches.

Beyond basic creation and destruction, Rudel supports point-in-time snapshots for easy rollback, reusable templates for standardized starting points, zip-based export and import for sharing environments, and configurable auto-cleanup for stale sandboxes. Everything is driven through WP-CLI, and every sandbox includes a `CLAUDE.md` file so AI coding agents have the context they need.

## Requirements

Rudel requires PHP 8.0 or later, WordPress 6.4 or later, and write access to `wp-config.php` for the initial bootstrap installation. If you plan to use the SQLite engine, you'll also need the PDO SQLite PHP extension, which is built into most PHP installations.
